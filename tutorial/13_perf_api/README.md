# Performance Modeling API

The perf model API includes the following commands in co-sim modes (shown with
example parameters):

```
hart=0 time=0  perf_model_fetch 1 0x80000000
hart=0 time=10 perf_model_decode 1
hart=0 time=20 perf_model_execute 1
hart=0 time=30 perf_model_retire 1
hart=0 time=40 perf_model_drain_store 1
hart=0 time=50 perf_model_flush 2
hart=0 time=60 perf_model_predict_branch 4 0 0x1000
```

The test.cmd file shows how these commands might be used. This example command
log file was generated by a script which automaticaly scrambles the fetch,
decode, execute, etc. steps in such a way that it does not violate any
dependencies.

In this example, you can see that the following instructions were reordered
because their execute step comes before the execute of an order instruction:

- 6 is executed before 5
- 18 is executed before 17
- 22 is executed before 21

```
hart=0 time=20 perf_model_execute 6
hart=0 time=21 perf_model_retire 4
hart=0 time=22 perf_model_fetch 8 0x80000024 # c.sub    x10, x11
hart=0 time=23 perf_model_execute 5
# ...
hart=0 time=65 perf_model_fetch 18 0x80000018 # beq      x10, x11, . + 0x10
hart=0 time=66 perf_model_retire 16
hart=0 time=67 perf_model_fetch 19 0x8000001c # blt      x11, x10, . + 0x8
hart=0 time=68 perf_model_decode 18
hart=0 time=69 perf_model_execute 18
hart=0 time=70 perf_model_decode 19
hart=0 time=71 perf_model_execute 17
# ...
hart=0 time=81 perf_model_fetch 22 0x80000018 # beq      x10, x11, . + 0x10
hart=0 time=82 perf_model_decode 22
hart=0 time=83 perf_model_execute 22
hart=0 time=84 perf_model_retire 20
hart=0 time=85 perf_model_execute 21
```
